Priority 1: Zero-Coverage Modules

These tools address Qualys modules with zero existing coverage. Each tool unlocks an entire category of customer questions.

get_webapp_vulns Critical — 50 questions

Returns web application vulnerability findings from the Qualys WAS module, including OWASP categories, severity, URL, and remediation guidance.

Parameters: webapp_id (optional), severity (optional filter: 1-5), status (optional: NEW, ACTIVE, REOPENED, FIXED), limit (default 100)

API endpoint: POST /qps/rest/3.0/search/was/finding

Return shape:

{
  "webapp_id": "12345",
  "total_findings": 42,
  "by_severity": {"5": 2, "4": 8, "3": 12, "2": 15, "1": 5},
  "findings": [
    {
      "id": "67890",
      "qid": 150001,
      "title": "SQL Injection",
      "severity": 5,
      "owasp": "A03:2021 Injection",
      "url": "https://app.example.com/login",
      "status": "ACTIVE",
      "first_detected": "2026-01-15",
      "times_detected": 3
    }
  ]
}

get_expiring_certs Critical — 30 questions

Returns certificates expiring within a specified window from Qualys CertView, enabling proactive renewal tracking and outage prevention.

Parameters: days (default 30, certificates expiring within this many days), grade (optional filter: A, B, C, D, F), limit (default 100)

API endpoint: POST /certview/v2/certificates

Return shape:

{
  "expiring_within_days": 30,
  "total": 18,
  "by_grade": {"A": 5, "B": 8, "C": 3, "F": 2},
  "certificates": [
    {
      "id": "cert-abc123",
      "subject": "*.example.com",
      "issuer": "DigiCert",
      "expires": "2026-04-01",
      "days_remaining": 16,
      "grade": "B",
      "key_size": 2048,
      "hosts": ["web1.example.com", "web2.example.com"]
    }
  ]
}

get_compliance_posture High — 45 questions

Priority 2: High-Impact Extensions

These tools extend partially covered modules with high question counts.

get_scan_status — 14 questions

Returns the status of vulnerability scans (running, queued, paused, finished, failed) from the VMDR scan API. Enables scan monitoring and troubleshooting workflows.

Parameters: scan_id (optional), status (optional filter), days (default 7), limit (default 50)

API endpoint: POST /api/2.0/fo/scan/

get_asset_inventory — 35 questions

Returns a filterable, paginated asset inventory from CSAM v2 with grouping by OS, location, tag, or business unit. Answers bulk asset questions that get_asset (single-asset) cannot.

Parameters: filter (QQL string), group_by (optional: os, tag, location), limit (default 100), offset (default 0)

API endpoint: POST /rest/2.0/search/am/asset

get_vuln_trends — 20 questions

Computes vulnerability trend data over time by comparing detection snapshots. Shows whether the environment is improving or degrading across severity levels.

Parameters: days (default 30), severity (optional filter), group_by (optional: severity, asset_tag)

API endpoint: Computed from VMDR detection API snapshots

get_vuln_exceptions — 10 questions

Returns vulnerability exceptions (accepted risks, false positives, disabled QIDs) to provide visibility into what has been explicitly excluded from remediation tracking.

Parameters: status (optional: active, expired), qid (optional), limit (default 100)

API endpoint: GET /api/2.0/fo/exception/

Priority 3: Valuable Extensions

Lower-priority tools that round out coverage for modules already partially addressed.

  • get_cloud_inventory — 15 questions. Lists cloud resources across AWS, Azure, and GCP using TotalCloud connectors API. Provides resource counts, types, and regions.
  • get_cloud_compliance — 10 questions. Returns cloud compliance evaluation results (CIS Benchmarks, custom policies) from TotalCloud evaluations API.
  • get_container_inventory — 10 questions. Lists container images, registries, and running containers with vulnerability summary from the Container Security API.
  • get_webapp_inventory — 10 questions. Lists web applications tracked by WAS with last scan date, authentication status, and finding counts.
