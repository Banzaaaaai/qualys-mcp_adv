Threat Intelligence Tools

Real-time threat indicators for ransomware, active exploits, CISA KEV, and more. Filter vulnerabilities by threat category to prioritize what matters most.

get_threat_intel

Search the Qualys Knowledge Base for newly published vulnerabilities filtered by Real-Time Threat Indicators (RTI). This tool surfaces vulnerabilities that carry specific threat characteristics — ransomware linkage, active exploitation, public exploit code, and more — so you can focus remediation on the threats that matter most.

The threat_type parameter accepts any of the 12 RTI categories below. When omitted, all threat-tagged vulnerabilities are returned.

RTI Categories

Category Description
RansomwareVulnerabilities associated with known ransomware campaigns
MalwareVulnerabilities exploited by malware families
Active_AttacksCurrently being exploited in the wild
Exploit_PublicPublic exploit code available (Metasploit, PoC, etc.)
Easy_ExploitLow complexity exploitation
WormableCan propagate without user interaction
Cisa_Known_Exploited_VulnsOn the CISA Known Exploited Vulnerabilities catalog
Denial_of_ServiceCan cause service disruption
Privilege_EscalationAllows elevation of privileges
Remote_Code_ExecutionAllows remote code execution
Predicted_High_RiskML-predicted high risk based on vulnerability characteristics
Unauthenticated_ExploitationExploitable without credentials

Parameters

Parameter Type Default Description
threat_type str "" RTI category to filter by (see table above). Empty string returns all threat-tagged vulns.
days int 30 How far back to search for published vulnerabilities.
limit int 200 Maximum number of vulnerabilities to return.

Example Questions

Natural Language Question Tool Call
"Are there any new ransomware-linked vulnerabilities?" get_threat_intel(threat_type="Ransomware", days=30)
"What vulnerabilities are being actively exploited right now?" get_threat_intel(threat_type="Active_Attacks", days=7)
"Show me CISA KEV additions from the past week" get_threat_intel(threat_type="Cisa_Known_Exploited_Vulns", days=7)
"Any wormable vulnerabilities published recently?" get_threat_intel(threat_type="Wormable", days=14)
"Show me vulns with public exploit code" get_threat_intel(threat_type="Exploit_Public", days=30)
"Are there easy-to-exploit vulnerabilities this month?" get_threat_intel(threat_type="Easy_Exploit", days=30)
"What remote code execution vulns were published this week?" get_threat_intel(threat_type="Remote_Code_Execution", days=7)
"Show me all threat-tagged vulnerabilities from the last 60 days" get_threat_intel(days=60, limit=200)

get_cdr_findings

Cloud Detection & Response findings from TotalCloud. Detects malware, command-and-control (C2) activity, crypto-miners, lateral movement, and other threats in cloud workloads across AWS, Azure, and GCP.

Parameters

Parameter Type Default Description
days int 7 Look-back window in days.
severity str "" Filter by severity — CRITICAL, HIGH, MEDIUM, LOW. Empty returns all.
cloud_provider str "" Filter by cloud provider — AWS, Azure, GCP. Empty returns all.

Examples

Natural Language Question Tool Call
"What cloud threats were detected this week?" get_cdr_findings(days=7)
"Show me critical AWS CDR findings" get_cdr_findings(severity="CRITICAL", cloud_provider="AWS")
"Are there crypto-mining activities in our cloud?" get_cdr_findings(days=30)

get_edr_events

Endpoint Detection and Response events — malware detections, ransomware behavior, C2 callbacks, process injection, lateral movement, and suspicious process executions. Filter by severity, category, or specific host for targeted investigation.

Parameters

Parameter Type Default Description
days int 7 Look-back window in days.
severity str "" Filter by severity — CRITICAL, HIGH, MEDIUM, LOW. Empty returns all.
category str "" Filter by event category substring (e.g. Malware, C2, LateralMovement).
host str "" Filter by hostname substring.
limit int 50 Maximum events to return.

Expected Output Shape

{
  "summary": {
    "total": 42,
    "critical": 3,
    "high": 12,
    "medium": 18,
    "low": 9,
    "affectedHosts": 7
  },
  "byCategory": {
    "Malware": 15,
    "C2": 8,
    "SuspiciousProcess": 19
  },
  "topHosts": [
    { "hostname": "web-prod-01", "eventCount": 12 }
  ],
  "events": [
    {
      "id": "evt-001",
      "severity": "CRITICAL",
      "category": "Malware",
      "name": "Trojan.GenericKD",
      "hostname": "web-prod-01",
      "ip": "10.0.1.25",
      "user": "svc-app",
      "process": "powershell.exe",
      "timestamp": "2026-03-15T08:23:41Z"
    }
  ]
}

Examples

Natural Language Question Tool Call
"What malware was detected this week?" get_edr_events(days=7)
"Show me critical endpoint threats" get_edr_events(severity="CRITICAL")
"Are any hosts showing ransomware behavior?" get_edr_events(category="ransomware", days=30)

get_fim_events

File Integrity Monitoring events — unauthorized file changes, critical system file modifications, registry changes, and suspicious path activity. Includes off-hours flagging for change activity outside normal business windows.

Parameters

Parameter Type Default Description
days int 1 Look-back window in days.
severity str "" Filter by severity — CRITICAL, HIGH, MEDIUM, LOW. Empty returns all.
host str "" Filter by hostname substring.
path str "" Filter by file path prefix (e.g. /etc/, C:\Windows\System32).
limit int 100 Maximum events to return.

Expected Output Shape

{
  "summary": {
    "total": 28,
    "critical": 2,
    "high": 5,
    "affectedHosts": 4,
    "modified": 20,
    "created": 5,
    "deleted": 3
  },
  "topHosts": [
    { "hostname": "db-prod-03", "eventCount": 9 }
  ],
  "criticalChanges": [
    {
      "path": "/etc/passwd",
      "hostname": "db-prod-03",
      "action": "MODIFIED",
      "timestamp": "2026-03-15T03:14:22Z",
      "offHours": true
    }
  ],
  "events": [
    {
      "action": "MODIFIED",
      "path": "/etc/sudoers",
      "hostname": "db-prod-03",
      "timestamp": "2026-03-15T03:14:22Z",
      "severity": "CRITICAL",
      "user": "root",
      "offHours": true
    }
  ]
}

Examples

Natural Language Question Tool Call
"What file changes happened today?" get_fim_events(days=1)
"Were /etc/passwd or sudoers modified?" get_fim_events(path="/etc/passwd")
"Show me FIM events on production servers" get_fim_events(host="prod")
