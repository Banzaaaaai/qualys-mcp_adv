Operations Tools

Daily operational tools for morning reports, scanner health monitoring, scan management, patch status, TruRisk Eliminate, and security program coaching.

get_morning_report

Cross-module daily security briefing. New vulns, ransomware/exploit flags, top risks, and action items. Supports quick=True for a fast environment snapshot.

Parameters

NameTypeDefaultDescription
quickboolFalseReturn a fast environment snapshot instead of the full briefing

Examples

You askTool call
"What happened overnight?"get_morning_report()
"Quick health check — assets, OS mix, cloud vs on-prem?"get_morning_report(quick=True)

get_weekly_priorities

Top risk assets ranked by TruRisk for the week. Helps teams focus remediation effort where it matters most.

Parameters

NameTypeDefaultDescription
limitint10Number of top-risk assets to return

Examples

You askTool call
"What should my team focus on this week?"get_weekly_priorities()

get_security_posture

Overall security health — health score, risk distribution, container and cloud stats.

Examples

You askTool call
"How secure are we?"get_security_posture()
"Give me an executive summary of our security posture"get_security_posture()

get_scanner_health

Scanner appliance status — online/offline, failed scans, capacity, and signature updates.

Examples

You askTool call
"Are all my scanners healthy?"get_scanner_health()
"Which scanners are offline?"get_scanner_health()

get_scan_status

VM scan status and history — running, queued, completed, and failed scans.

Parameters

NameTypeDefaultDescription
statestr"Running,Paused,Queued,Error"Comma-separated scan states to filter
daysint7Lookback window in days
limitint50Maximum number of scans to return

Examples

You askTool call
"Show me all running scans"get_scan_status(state="Running")
"What scans failed in the last 24 hours?"get_scan_status(state="Error", days=1)
