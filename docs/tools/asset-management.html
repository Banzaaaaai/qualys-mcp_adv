Asset Management Tools

Tools for asset risk assessment, inventory search, tech debt tracking, environment snapshots, and tag-based risk analysis.

get_asset_risk

Single-asset risk profile — TruRisk score, OS, criticality, software inventory, and EOL status. Provides a quick summary view of an individual asset's risk posture.

Parameters

Parameter Type Default Description
asset_id str required CSAM asset ID from any tool that returns asset lists.

Examples

Natural Language Question Tool Call
"Why is asset 233946644 showing up as high risk?" get_asset_risk(asset_id="233946644")

get_asset_full_profile

Full single-asset profile combining CSAM + ETM + VMDR detections in parallel. Returns comprehensive risk data including software inventory, vulnerability detections, ETM findings, tags, and summary counts. Performance is approximately 5–8 seconds cold, 2 seconds warm.

Parameters

Parameter Type Default Description
asset_id str required CSAM asset ID.
detail str "full" Detail level — "summary" (fast, ~2s) or "full" (complete, ~6s).

Examples

Natural Language Question Tool Call
"Give me everything on asset 233946644" get_asset_full_profile(asset_id="233946644")

get_risk_by_tag

Aggregate risk distribution for a tagged asset group. Returns total asset count, TruRisk tier distribution (critical >900, high >700, elevated >500), top risky assets ranked by score, and EOL count scoped to the tag. Performance is approximately 3 seconds (parallel CSAM count queries).

Parameters

Parameter Type Default Description
tag str required Tag name to filter by (e.g. PCI, Production, DMZ, AWS, HIPAA).
limit int 20 Maximum number of top-risk assets to return.

Examples

Natural Language Question Tool Call
"How's our Production environment risk-wise?" get_risk_by_tag(tag="Production")
"What's the risk exposure in our PCI scope?" get_risk_by_tag(tag="PCI")

get_environment_summary

Fast all-CSAM environment snapshot: OS breakdown, cloud provider split, EOL counts, and criticality tier distribution. Completes in under 3 seconds. Takes no parameters.

Examples

Natural Language Question Tool Call
"What does our environment look like?" get_environment_summary()

get_asset_inventory

Asset inventory search powered by CSAM. Find assets by OS, tag, free-text query, EOL status, or last-seen date. Supports listing all available tags and asset groups for discovery.

Parameters

Parameter Type Default Description
query str "" Free-text search on hostname or asset name.
tag str "" Filter by asset tag name.
os str "" Filter by OS (e.g. Windows, Linux, Ubuntu, CentOS).
days_since_seen int 0 Only assets NOT seen in the last N days (stale assets). 0 disables filter.
eol_only bool False Only return end-of-life assets.
limit int 50 Maximum results to return.

Expected Output Shape

{
  "total": 1247,
  "assets": [
    {
      "id": "233946644",
      "name": "web-prod-01.example.com",
      "ip": "10.0.1.25",
      "os": "Windows Server 2022",
      "lastSeen": "2026-03-15",
      "tags": ["Production", "PCI"],
      "truRiskScore": 875,
      "openVulns": 12,
      "eolStatus": null
    }
  ],
  "summary": {
    "total": 1247,
    "returned": 50,
    "byOS": {
      "Windows Server 2022": 340,
      "Ubuntu 22.04": 280,
      "CentOS 7": 150
    },
    "byTag": {
      "Production": 500,
      "Development": 400,
      "PCI": 120
    },
    "eolCount": 47
  }
}

Examples

Natural Language Question Tool Call
"Show me all Windows assets" get_asset_inventory(os="Windows")
"Which assets haven't been seen in 90 days?" get_asset_inventory(days_since_seen=90)
"Show me all EOL assets" get_asset_inventory(eol_only=True)

get_tech_debt

End-of-life and end-of-support systems — OS and hardware assets running unsupported software, sorted by criticality and risk score. Use for tech debt assessments, EOL/EOS exposure audits, and upgrade planning.

Examples

Natural Language Question Tool Call
"How many EOL systems do we have?" get_tech_debt()

get_cloud_risk

Cloud security posture across AWS, Azure, and GCP. Returns connected accounts, CIS benchmark control failures, and CDR threat findings. Set include_threats=True to include detailed Cloud Detection & Response threat data.

Parameters

Parameter Type Default Description
include_threats bool False Include detailed CDR threat findings. Set False for posture-only.
days int 30 CDR look-back window in days. Only used when include_threats=True.
severity str "" Filter by severity — CRITICAL, HIGH, MEDIUM, LOW. Empty returns all.
cloud_provider str "" Filter by cloud provider — AWS, Azure, GCP. Empty returns all.

Examples

Natural Language Question Tool Call
"What's our cloud posture?" get_cloud_risk()
"What cloud threats were detected this week?" get_cloud_risk(include_threats=True, days=7)

get_image_vulns

Container image vulnerabilities — severity breakdown and individual vulnerability details with fix versions. Use for pre-deployment image scanning review and container remediation planning.

Parameters

Parameter Type Default Description
image_id str required TotalCloud image ID (from get_asset_inventory or get_weekly_priorities container risk section).
limit int 50 Maximum vulnerabilities to return.

Examples

Natural Language Question Tool Call
"What vulns are in our production container images?" get_image_vulns(image_id="abc123")
